# Client Communication Standards

A mandatory formatting framework for client-facing reports that follows Background, Analysis, Conclusions structure with required data source attribution for every finding.

**The pain point:** Client reports that jump straight to recommendations without showing the work lose trust. Reports with vague statements like "performance declined" without specifying what data, what time period, and how much leave clients with more questions than answers. This skill forces rigorous data transparency into every report.

---

## What's Inside

- Strict 3-section framework: Background, Analysis, Conclusions (no more, no fewer)
- "What We Looked At" requirement for every finding — must specify data source, time period, and methodology
- "Finding" statement templates that tie data to conclusions
- Conclusions subsections: What's Happening, What We're Doing About It, Expected Timeline, Key Takeaways
- Good vs. bad examples for each section
- Format variations for investigation summaries, monthly reviews, and strategy presentations
- Quality checklist for pre-delivery review
